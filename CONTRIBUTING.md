## ğŸš€ Getting Started

### 1. Fork the Repository
Click the "Fork" button at the top right of the repository page.

### 2. Clone Your Fork
```bash
git clone https://github.com/YOUR_USERNAME/qbitsonly.git
cd qbitsonly
```

### 3. Create a Branch
```bash
git checkout -b feature/your-feature-name
# or
git checkout -b fix/bug-description
```

### 4. Make Your Changes
- Edit the files
- Test locally (open `index.html` or use local server)
- Ensure everything works on mobile too

### 5. Commit Your Changes
```bash
git add .
git commit -m "Add: Brief description of your changes"
```

Use descriptive commit messages:
- `Add: New lesson on quantum entanglement`
- `Fix: Navigation menu on mobile devices`
- `Improve: Explanation of superposition in lesson 3`
- `Update: Broken link in README`

### 6. Push to Your Fork
```bash
git push origin feature/your-feature-name
```

### 7. Create a Pull Request
- Go to the original repository
- Click "New Pull Request"
- Select your fork and branch
- Describe your changes clearly
- Submit!

## ğŸ“ Content Guidelines

### Writing Lessons

1. **Clarity First**
   - Explain concepts simply
   - Avoid unnecessary jargon
   - Define technical terms
   - Use analogies when helpful

2. **Structure**
   - Start with a clear introduction
   - Break into logical sections (h2, h3)
   - Include practical examples
   - End with key takeaways
   - Add navigation to previous/next lessons

3. **Formatting**
   - Use info boxes for important concepts
   - Include code examples where relevant
   - Add visualizations (SVG/CSS preferred)
   - Keep paragraphs short and readable

4. **Example Lesson Template**
```html
<h2>Introduction</h2>
<p>Brief overview of the topic...</p>

<div class="info-box">
```

### Style Guide

**HTML:**
- Use semantic HTML5 elements
- Include proper meta tags
- Ensure accessibility (alt text, ARIA labels)
- Test responsive design

**CSS:**
- Follow existing color scheme (CSS variables)
- Use consistent spacing
- Test on multiple screen sizes
- Maintain dark theme aesthetic

**JavaScript:**
- Keep it simple and readable
- Add comments for complex logic
- Ensure progressive enhancement
- Test in multiple browsers

## ğŸ¨ Design Principles

1. **Readability:** Text should be easy to read with good contrast
2. **Simplicity:** Don't overcomplicate the interface
3. **Consistency:** Follow established patterns
4. **Accessibility:** Everyone should be able to learn
5. **Performance:** Keep it fast and lightweight
